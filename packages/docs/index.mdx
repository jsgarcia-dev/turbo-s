## Testando a API

Como os componentes de playground interativo na documentação requerem uma configuração avançada, você tem várias opções para testar a API:

1. **Swagger UI**: [http://localhost:3001/api/docs](http://localhost:3001/api/docs)
2. **Postman/Insomnia**: Importe a especificação [OpenAPI](http://localhost:3001/api/docs-json)
3. **cURL**: Use os exemplos fornecidos em cada página de endpoint

### Pré-requisitos

Para usar o Swagger UI, você precisa:

```bash
# Navegar até a pasta da API
cd apps/api

# Iniciar o servidor da API
npm run dev
```

Depois, acesse [http://localhost:3001/api/docs](http://localhost:3001/api/docs) no navegador.
